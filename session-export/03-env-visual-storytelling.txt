# Environmental Visual Storytelling Review — WraithWall Public Surface

**Scope:** Public, unauthenticated pages on `wraithwall.online`  
**Constraint:** Foreground always wins. Current brand is *premium editorial paper* (warm cream, `#FAFAF7`) with a *dark ops terminal* as the hero focal point — not generic neon cyber.

**Global principle:** Use environmental backgrounds sparingly and *by narrative tier*. WraithWall already has cinematic identity in the HeroTerminal + AttackGlobe. Full-page environments should extend that story, not compete with it.

---

## Tier system (apply everywhere)

| Tier | Treatment | Where |
|------|-----------|-------|
| **A — Cinematic hero only** | Environment behind hero band; 60–75% scrim into paper below | Landing, product landings |
| **B — Atmospheric wash** | 5–12% opacity, fixed or slow parallax, never behind body text | Marketing shells, launch |
| **C — Texture only** | Grain, vignette, blueprint linework (already in theme) | Docs, blog, knowledge |
| **D — Tactical minimal** | Grid, scanline, no scenery | Gateway, auth |
| **E — None** | Pure paper | Tools, legal, forms |

## 1. Landing `/` (Vue SSG)

| Question | Answer |
|----------|--------|
| **Environmental background?** | **Yes — hero only** |
| **Why?** | This is the brand moment. You already have a dark terminal + globe; the environment should *frame* that as a window into live operations, not wallpaper the whole page. |
| **Best environment** | **Orbital intelligence command** — Earth curvature at extreme distance, faint satellite arcs, soft red telemetry dots matching `#C41A1A`. Feels like watching sensors from above, not a movie spaceship interior. |
| **Static or animated?** | **Subtle animation** — slow orbital drift (2–3px/min parallax), occasional blink on arc nodes. Globe/feed stay the motion hero. |
| **Emotion vs distraction?** | **Improves emotion** if confined to hero. Full-page would fight readability and the paper editorial system. |
| **Implementation** | **Layered parallax** (2 layers: distant Earth haze + arc SVG) behind hero; **lightweight Canvas** only if arcs tie to existing AttackGlobe coordinates. **No video loop** (perf + mobile). **No AI full-bleed** behind stat cards. |

**Below-the-fold sections** (capabilities, feeds, architecture gallery): **Tier C only** — keep paper; diagrams already provide visual weight.

# Cross-cutting recommendations

## Where environments help most (priority order)

1. **Landing hero** — orbital command framing existing globe/terminal  
2. **BGP + Canary product landings** — distinct product stories  
3. **Gateway/auth shell** — shared tactical language  
4. **Architecture page header** — blueprint vault texture  
5. **Blog index header** — observatory wash  

## Where environments hurt most (avoid)

- Knowledge articles, blog posts, docs hub, privacy, contact, tools, thank-you pages  
- Any full-page dark environment on light-theme reading pages  
- Video loops anywhere (mobile, accessibility, brand restraint)  
- Generic “matrix rain,” neon cityscapes, hooded hackers — off-brand for WraithWall’s editorial + live-sensor identity  

## WraithWall-specific visual language (not generic cyber)

| Motif | Use for |
|-------|---------|
| Orbital sensor lattice | Landing, global monitoring narrative |
| Route arcs / AS paths | BGP only |
| Tripwire / vault corridor | Canary only |
| Blueprint linework | Architecture, launch diagrams context |
| Paper grain + warm vignette | All long-form reading (already in theme) |
| Dark terminal window | Hero focal object — not replaced by background art |

## Technical implementation hierarchy

| Method | Best for | Avoid for |
|--------|----------|-----------|
| **CSS gradient + vignette** | Knowledge, blog posts, auth | — |
| **Animated SVG** | Gateway radar, BGP arcs, architecture grid | Long-form body |
| **Layered parallax (CSS)** | Landing hero, product landings | Forms, articles |
| **Lightweight Canvas** | Extending AttackGlobe only | Static pages |
| **AI-generated hero artwork** | Landing hero, blog index, 404, product landings (hero band) | Full page, text columns |
| **Video loop** | — | Everywhere on public site |
| **No background** | Docs, knowledge body, tools, legal, contact | — |

## Readability safeguards (non-negotiable)

- Minimum **70% opaque scrim** between environment and any text block  
- Environment never behind paragraphs — only behind hero bands or page headers  
- Cards/panels stay `#FFFFFF` / paper with existing borders  
- Respect `prefers-reduced-motion`: static fallback for all animated environments  
- Dark environments only in **gateway, auth, terminal components** — public marketing stays paper-first  

---

## Summary verdict

WraithWall should **not** become an “environment-first” site. It should become a **paper-first intelligence publication** with **surgical cinematic environments** at trust boundaries (gateway/auth), **product identity** (canary/bgp), and **the hero moment** (landing). That matches what you already built: live terminal as the window, cream as the reading room.

**Highest ROI, lowest risk:** Landing hero parallax + gateway SVG depth + architecture blueprint header.  
**Highest risk:** Full-page AI environments on docs, blog, or knowledge articles — would break the premium editorial identity and hurt readability.
