# Support

Thanks for using WraithWall. This page explains where to get help and what to expect.

WraithWall is a **solo-maintained** open-source project. Support is best-effort.

## Getting help

| I want to… | Where to go |
| ---------- | ----------- |
| Ask a question / discuss an idea | [GitHub Discussions](https://github.com/niffyhunt/wraithwall/discussions) (or open an issue if Discussions are unavailable) |
| Report a bug | [Open a bug report](https://github.com/niffyhunt/wraithwall/issues/new/choose) |
| Request a feature | [Open a feature request](https://github.com/niffyhunt/wraithwall/issues/new/choose) |
| Report a **security vulnerability** | **Do not open a public issue.** Follow [SECURITY.md](SECURITY.md) → `contact@wraithwall.online` |
| Contribute code | Read [CONTRIBUTING.md](CONTRIBUTING.md) first |

## Before opening an issue

1. Search [existing issues](https://github.com/niffyhunt/wraithwall/issues) — it may already be reported.
2. Check [README.md](README.md), [AGENTS.md](AGENTS.md), and the `docs/` directory.
3. Include: what you expected, what happened, steps to reproduce, and your environment (OS, Python version, deployment mode).

## Response expectations

| Item | Commitment |
| ---- | ---------- |
| Security reports | Human acknowledgement within **72 hours** (see [SECURITY.md](SECURITY.md)) |
| Bug reports / questions | Best-effort — typically days to weeks |
| Pull requests | Reviewed as time allows; small, focused PRs merge fastest |

## Not supported

- Custom deployment consulting or one-on-one setup support
- The separate `breach-monitor/`, `cowrie-analyzer/`, and `ops-dashboard/` subprojects are provided as-is
- Third-party services (hosting, DNS, payment/intel providers)
